# Понимание пропенсити, стохастических политик и value моделей

Этот мини-туториал объясняет ключевые термины из обучения с подкреплением и связанных областей. После прочтения вы будете понимать:

* что такое **propensity**;
* что означает **нулевой propensity** и почему это проблема;
* что такое **стохастическая политика выбора**;
* что такое **value модель** (функция ценности).

## 1. Propensity (вероятность выбора действия)

**Propensity** – это вероятность того, что политика выберет определённое действие в заданном состоянии. Политику \(\pi(a\mid s)\) можно рассматривать как распределение вероятностей по действиям `a` при условии состояния `s`. Поэтому propensity иногда называют "propensity score" или просто \(\pi(a\mid s)\).

### Пример

В стратегии ε-greedy агент с вероятностью \(1-\varepsilon\) выбирает действие с максимальным ожидаемым вознаграждением и с вероятностью \(\varepsilon\) случайно выбирает любое действие. Для каждого действия `a` можно выписать propensity:

\[
\pi(a\mid s) =
\begin{cases}
1-\varepsilon + \varepsilon/|A|, & a = \text{лучшее действие},\\
\varepsilon/|A|, & \text{иначе},
\end{cases}
\]

где \(|A|\) – количество возможных действий.

Propensity важен в off-policy оценке: если мы собрали данные под одной политикой, а хотим оценить другую, мы используем веса вида \(\frac{\pi_{\text{target}}(a\mid s)}{\pi_{\text{behavior}}(a\mid s)}\).

## 2. Нулевой propensity

**Нулевой propensity** возникает, когда политика никогда не выбирает определённое действие в некотором состоянии, то есть \(\pi(a\mid s) = 0\). Это проблема для off-policy методов: если наблюдаемая политика не выполняла действие, которое нас интересует, значение веса \(\frac{\pi_{\text{target}}}{\pi_{\text{behavior}}}\) становится бесконечным, и оценка обрывается.

### Почему это важно

* **Невозможно оценить новое действие** – мы просто не имеем данных, чтобы понять, что будет при выборе `a`.
* **Чувствительность к разреженным данным** – даже очень маленький propensity ведёт к большим весам и высокой дисперсии оценок.

### Как избегать нулевого propensity

* Вводить минимальный уровень рандомизации (ε > 0).
* Применять дополнительные методы, например, "clip" весов или комбинировать модель‑based подход.

## 3. Стохастическая политика выбора

**Стохастическая политика** генерирует распределение вероятностей по действиям и выбирает действие случайно в соответствии с этим распределением. В противоположность ей **детерминированная** политика всегда выдаёт одно и то же действие для данного состояния.

### Зачем нужна стохастичность

* **Исследование**. Агент не зацикливается на одном поведении и может находить более выгодные стратегии.
* **Стабильность обучения**. В методах policy gradient стохастическая политика даёт гладкую функцию потерь.
* **Робастность**. В задачах с неопределённостью смешение действий может быть выгоднее жёсткого выбора.

### Пример

В непрерывном пространстве действий actor‑critic политика может выдавать параметры нормального распределения \(\mathcal N(\mu, \sigma^2)\). Выбор конкретного действия – сэмплирование из этого распределения.

## 4. Value модель

**Value модель** (или функция ценности, value function) оценивает ожидаемое суммарное вознаграждение от состояния или от пары состояние‑действие.

* **State-value function** \(V^\pi(s)\) – ожидаемая награда, которую получит агент, начиная в состоянии `s` и следуя политике \(\pi\):

\[V^\pi(s) = \mathbb{E}_\pi \left[ \sum_{t=0}^\infty \gamma^t r_{t} \mid s_0 = s \right]\]

* **Action-value function** \(Q^\pi(s,a)\) – ожидаемая награда, если начать в `s`, выполнить `a` и далее следовать \(\pi\):

\[Q^\pi(s,a) = \mathbb{E}_\pi \left[ \sum_{t=0}^\infty \gamma^t r_{t} \mid s_0 = s, a_0 = a \right]\]

### Для чего нужна value модель

* **Оценка политики** – понимаем, насколько хорошо текущая стратегия.
* **Выбор действий** – жадные стратегии выбирают действие с максимальным \(Q\).
* **Обучение** – многие алгоритмы (SARSA, DQN, actor‑critic) строят и обновляют приближение value функции.

## 5. Связь между терминами

* Propensity – вероятность выбора действия; стохастическая политика определяет эти вероятности.
* Нулевой propensity показывает, что некоторые действия политика никогда не пробует, что затрудняет оценку и улучшение.
* Value модель опирается на действия, которые мы хотя бы иногда выбираем; без достаточной propensity оценка \(Q\) или \(V\) для этих действий невозможна.

## 6. Мини‑пример на Python

```python
import numpy as np

# стохастическая политика ε-greedy
class EpsilonGreedyPolicy:
    def __init__(self, q_values, epsilon=0.1):
        self.q = q_values  # оценка value
        self.eps = epsilon

    def act(self):
        if np.random.rand() < self.eps:
            return np.random.randint(len(self.q))  # случайное действие
        return int(np.argmax(self.q))  # жадный выбор

    def propensity(self, a):
        n = len(self.q)
        best = np.argmax(self.q)
        if a == best:
            return 1 - self.eps + self.eps / n
        else:
            return self.eps / n

q = [1.2, 0.5, 0.7]  # простая value модель Q(s, a)
policy = EpsilonGreedyPolicy(q, epsilon=0.2)

for a in range(len(q)):
    print(f"propensity для действия {a}: {policy.propensity(a):.2f}")
```

Этот код вычисляет propensity для каждого действия в ε-greedy политике, опираясь на value модель `q`.

## Итоги

* **Propensity** – вероятность выбора действия политикой.
* **Нулевой propensity** – действие никогда не выбирается, что делает невозможной его оценку.
* **Стохастическая политика** задаёт распределение вероятностей по действиям и обеспечивает исследование.
* **Value модель** оценивает ожидаемое вознаграждение и используется для сравнения действий.

Эти концепции взаимосвязаны: политика (детерминированная или стохастическая) определяет propensity; value модель помогает выбирать действия; нулевой propensity препятствует оценке и улучшению стратегии.
